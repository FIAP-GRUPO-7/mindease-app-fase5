# MindEase — Responsabilidades da Equipe

Este documento define as responsabilidades e os limites de atuação de cada membro da equipe,
garantindo um fluxo de projeto organizado, sustentável e sem conflitos.

---

## 1. Henrique — Fundação do Projeto & Design System

Responsabilidades:
- Definição do conceito do produto  
- Definição do fluxo do usuário  
- Arquitetura do sistema  
- Decisões sobre a stack de tecnologia  
- Modelagem do domínio (entidades e casos de uso)  
- Fundação do design system  

Ponto de parada:
- Antes da Camada de Aplicação  

---

## 2. Diego — Camada de Aplicação & Orquestração

Responsabilidades:
- Serviços da camada de aplicação  
- Orquestração dos casos de uso  
- Implementação de repositórios  
- Ponte entre o domínio e a interface (UI)  

---

## 3. Kauane — Interface Web (UI/UX)

Responsabilidades:
- Implementação da interface web  
- Aplicação do design system  
- Implementação de acessibilidade  
- Coerência do fluxo visual  

---

## 4. Manoel — Aplicação Mobile

Responsabilidades:
- Implementação da versão mobile  
- Adaptação do fluxo cognitivo  
- Comportamentos específicos da plataforma  

---

## 5. Alexa — Qualidade, Testes & Apresentação Final

Responsabilidades:
- Estratégia de testes  
- Validação de acessibilidade  
- Organização do repositório  
- Suporte no vídeo final e na documentação  

---

## Regras

- Nenhuma fase concluída pode ser modificada sem alinhamento com seu responsável.  
- Toda fase deve ser commitada e versionada antes da passagem para o próximo responsável.  
- A camada de domínio é imutável, salvo em casos de extrema necessidade.  


EN:
# MindEase — Team Responsibilities

This document defines the responsibilities and boundaries of each team member
to ensure a smooth, organized, and sustainable project flow.

---

## 1. Henrique — Project Foundation & Design System

Responsibilities:
- Product concept definition
- User flow definition
- System architecture
- Technology stack decisions
- Domain modeling (entities and use cases)
- Design system foundation

Stop point:
- Before the Application Layer

---

## 2. Diego — Application Layer & Orchestration

Responsibilities:
- Application services
- Use case orchestration
- Repository implementations
- Bridge between domain and UI

---

## 3. Kauane — Web Interface (UI/UX)

Responsibilities:
- Web UI implementation
- Design system application
- Accessibility implementation
- Visual flow coherence

---

## 4. Manoel — Mobile Application

Responsibilities:
- Mobile version implementation
- Cognitive flow adaptation
- Platform-specific behaviors

---

## 5. Alexa — Quality, Tests & Final Presentation

Responsibilities:
- Testing strategy
- Accessibility validation
- Repository organization
- Support on final video and documentation

---

## Rules

- No one modifies a completed phase without alignment with its owner.
- Every phase must be committed and versioned before handoff.
- The domain layer is immutable unless absolutely necessary.
