# MindEase — Fluxo do Usuário

Este documento descreve o fluxo completo do usuário no aplicativo MindEase,
desde a abertura do app até o encerramento de uma sessão.

---

## 1. Ponto de Entrada — Seleção do Estado Cognitivo

Ao abrir o aplicativo, o sistema **não** apresenta painéis,
estatísticas ou explicações.

O usuário é confrontado com uma única pergunta:

**“Como você está se sentindo agora?”**

Opções disponíveis:
- Baixa Energia
- Focado
- Alta Sensibilidade

Essa escolha define todo o comportamento da sessão.

---

## 2. Adaptação do Sistema

Após a seleção do estado cognitivo, o sistema se adapta automaticamente:

- Densidade da interface
- Ritmo visual
- Animações
- Alertas e notificações

O usuário **não** configura essas opções manualmente.  
O sistema assume essas decisões.

---

## 3. Modo de Execução Guiada

O usuário entra no modo de execução guiada.

O sistema apresenta:
- Uma única tarefa ou ação
- Elementos visuais mínimos
- Nenhuma prioridade concorrente

O objetivo é permitir que o usuário aja sem planejamento ou fadiga de decisão.

---

## 4. Progressão da Sessão

Durante a execução:
- Notificações são minimizadas ou desativadas
- Indicadores de tempo são sutis ou ocultos
- Transições entre etapas são calmas e previsíveis

O sistema pode sugerir pausas ou continuidade de forma suave, sem pressão.

---

## 5. Organização Opcional

Caso o usuário opte por organizar tarefas, o sistema oferece:
- Uma estrutura visual simplificada
- Poucas etapas
- Nenhuma priorização forçada

A organização é opcional e nunca bloqueia a execução.

---

## 6. Encerramento da Sessão

Quando o usuário finaliza ou sai da sessão:
- Nenhum resumo de desempenho é exibido
- Nenhuma métrica de produtividade é apresentada

O sistema encerra a sessão com uma mensagem neutra e tranquila.

A experiência termina sem pressão cognitiva.


EN:
# MindEase — User Flow

This document describes the complete user flow of the MindEase application,
from opening the app to ending a session.

---

## 1. Entry Point — Cognitive State Selection

When the user opens the application, the system does not present dashboards,
statistics, or explanations.

The user is asked a single question:

“How are you feeling right now?”

Available options:
- Low Energy
- Focused
- High Sensitivity

This choice defines the entire session behavior.

---

## 2. System Adaptation

After the cognitive state is selected, the system automatically adapts:

- Interface density
- Visual rhythm
- Animations
- Alerts and notifications

The user does not configure these settings manually.
The system assumes these decisions.

---

## 3. Guided Execution Mode

The user enters the guided execution mode.

The system presents:
- A single task or action
- Minimal visual elements
- No competing priorities

The goal is to allow the user to act without planning or decision fatigue.

---

## 4. Session Progression

During execution:
- Notifications are minimized or disabled
- Time indicators are subtle or hidden
- Transitions between steps are calm and predictable

The system may gently suggest pauses or continuation, without pressure.

---

## 5. Optional Organization

If the user chooses to organize tasks, the system offers:
- A simplified visual structure
- Few steps
- No forced prioritization

Organization is optional and never blocks execution.

---

## 6. Session End

When the user finishes or leaves the session:
- No performance summaries are shown
- No productivity metrics are displayed

The system closes the session with a neutral and calm message.

The experience ends without cognitive pressure.
