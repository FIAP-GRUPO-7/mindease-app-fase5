# MindEase — Design System

Este documento define os princípios visuais e de interação do MindEase,
com foco em acessibilidade cognitiva e redução da carga mental.

---

## 1. Princípios Fundamentais de Design

### 1.1 Calma Cognitiva
A interface deve reduzir o ruído visual e evitar estímulos desnecessários.  
Todo elemento precisa justificar sua presença.

### 1.2 Uma Coisa por Vez
O usuário nunca deve ser incentivado a focar em mais de uma ação principal
por tela.

### 1.3 Previsibilidade Acima de Surpresa
Layouts, transições e interações devem ser consistentes e esperados.  
Animações surpreendentes ou mudanças bruscas de layout devem ser evitadas.

### 1.4 Feedback Suave
O feedback deve ser sutil, calmo e não urgente.  
Nada de cores agressivas, sons invasivos ou movimentos bruscos.

---

## 2. Sistema de Cores

- Fundo Primário: Cinza neutro escuro ou cinza claro suave  
- Texto Primário: Alto contraste, nunca branco puro ou preto puro  
- Cor de Destaque: Usada com moderação para indicar foco ou ação principal  

**Regras:**
- No máximo uma cor de destaque por tela  
- Evitar cores saturadas  
- A cor deve apoiar o foco, não a decoração  

---

## 3. Tipografia

- Família tipográfica: Sans-serif, altamente legível  
- Tamanho base da fonte: Confortável para leitura  
- Altura de linha: Espaçamento generoso para reduzir densidade visual  

**Regras:**
- Evitar fontes decorativas  
- Limitar variações de peso  
- Blocos de texto devem ser curtos e escaneáveis  

---

## 4. Espaçamento e Layout

- Utilizar unidades de espaçamento consistentes  
- Priorizar espaço em branco ao invés de separadores  
- Layouts devem parecer leves e respiráveis  

**Regras:**
- Evitar telas congestionadas  
- Limitar o número de elementos visíveis  

---

## 5. Movimento e Transições

- Animações devem ser lentas e sutis  
- O movimento deve guiar, não entreter  
- Todas as animações devem ser opcionais ou removíveis  

**Regras:**
- Nenhuma animação em loop  
- Nenhuma transição abrupta  

---

## 6. Adaptação ao Estado Cognitivo

A interface se adapta com base no estado cognitivo do usuário:

- **Baixa Energia:** UI mínima, menos elementos, ritmo mais lento  
- **Foco:** Estrutura clara, orientação sutil  
- **Alta Sensibilidade:** Quase nenhum movimento, mudanças mínimas de contraste  

A UI nunca deve permitir sobrescrever manualmente esses comportamentos.

---

## Objetivo do Design System

O design system existe para proteger o usuário de sobrecarga cognitiva
e proteger o produto de inconsistência visual.


EN:
# MindEase — Design System

This document defines the visual and interaction principles of MindEase,
focused on cognitive accessibility and reduced mental load.

## 1. Core Design Principles

### 1.1 Cognitive Calm
The interface must reduce visual noise and avoid unnecessary stimulation.
Every element must justify its presence.

### 1.2 One Thing at a Time
The user should never be encouraged to focus on more than one primary action
per screen.

### 1.3 Predictability Over Surprise
Layouts, transitions, and interactions must be consistent and expected.
Surprising animations or layout shifts are avoided.

### 1.4 Gentle Feedback
Feedback should be subtle, calm, and non-urgent.
No aggressive colors, sounds, or movements.

## 2. Color System

- Primary Background: Neutral dark gray or soft light gray
- Primary Text: High contrast, never pure white or pure black
- Accent Color: Used sparingly to indicate focus or primary action

Rules:
- No more than one accent color per screen
- Avoid saturated colors
- Color is used to support focus, not decoration

## 3. Typography

- Font family: Sans-serif, highly legible
- Base font size: Comfortable reading size
- Line height: Generous spacing to reduce visual density

Rules:
- Avoid decorative fonts
- Limit font weight variations
- Text blocks must be short and scannable

## 4. Spacing and Layout

- Use consistent spacing units
- Prefer white space over separators
- Layouts should feel breathable, never dense

Rules:
- Avoid crowded screens
- Limit the number of visible elements

## 5. Motion and Transitions

- Animations must be slow and subtle
- Motion should guide, not entertain
- All animations must be optional or removable

Rules:
- No looping animations
- No sudden transitions

## 6. Cognitive State Adaptation

The interface adapts based on the user's cognitive state:

- Low Energy: Minimal UI, fewer elements, slower rhythm
- Focus: Clear structure, subtle guidance
- High Sensitivity: Almost no motion, minimal contrast changes

The UI must never override these behaviors manually.

## Design System Goal

The design system exists to protect the user from cognitive overload
and protect the product from visual inconsistency.
